# Changelog for version 2.1.165

## Summary

Version 2.1.165 (2.1.163 → 2.1.165) is a small patch with 100% structural similarity to its predecessor. The two meaningful behavioral changes are: conversation compaction now forwards your extended thinking configuration to the summarization step when the model supports it, and API requests made by nested subagents are now tagged with a `cc_is_subagent` marker in the billing attribution header.

## Improvements


### Extended Thinking Preserved During Conversation Compaction

What: When Claude compacts a long conversation (summarizing older turns to free up context window space), it previously always ran the summarization step with thinking disabled. Now, if your active model is listed in a server-side capability map as thinking-compatible, your thinking configuration is forwarded to that summarization call as well.

Details:
- Compatibility is determined by `clientDataCache?.cedar_lagoon`, a server-delivered object mapping model family identifiers to a boolean
- If the active `mainLoopModel` resolves to an entry that is `true` in that map, the user's `thinkingConfig` is passed through to the compaction summarizer; otherwise thinking remains disabled as before
- No user action is needed — takes effect automatically for supported models (e.g. Claude Sonnet 4.6 with extended thinking enabled)
- Improves the depth of reasoning the summarizer can apply when deciding what context to preserve across a compaction event

Evidence: Compaction model call now reads `hM8(K.options.mainLoopModel) ? K.options.thinkingConfig : { type: "disabled" }`, where `hM8` checks `clientDataCache?.cedar_lagoon` (search for `"cedar_lagoon"`)


### Subagent Requests Tagged in Billing Attribution Header

What: API requests sent when Claude Code is running as a nested subagent (not the top-level interactive session) now include `cc_is_subagent=true` in the `x-anthropic-billing-header` attached to every request.

Details:
- The main interactive session is now explicitly identified with `isMainSession: true` in its agent context object
- Any agent context where `agentType === "subagent"` and `isMainSession` is absent or false will receive the `cc_is_subagent=true` tag in its attribution header
- This is transparent to users — no visible behavior change in the CLI
- Allows Anthropic's billing backend to distinguish direct interactive usage from usage generated by nested agents (e.g. subagents spawned via the Agent SDK or `Task` tool)

Evidence: Attribution header builder now appends ` cc_is_subagent=true;` when `lA6(Y) && !Y.isMainSession` (search for `"cc_is_subagent=true;"`)
