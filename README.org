#+TITLE: Harness Investigations

* Overview

Investigations into how AI coding harnesses work internally — Claude Code, the
Claude Agent SDK, and OpenAI Codex — and how to use them effectively. We track
released versions, generate diffs and changelogs, and publish documentation
derived from reading the source.

The repo started as =claude-investigations= focused on Claude Code, then grew
to cover the Claude Agent SDK and Codex. The name reflects the broader scope.

* The Changelog System

=tools/sync.py= drives a multi-project pipeline that downloads each release,
diffs it against the previous release, asks an LLM to summarize the change,
and (optionally) posts the result to Discord forum threads.

** Tracked Projects

| Project          | Source                                | Notes                                     |
|------------------+---------------------------------------+-------------------------------------------|
| =claude-code=    | npm =@anthropic-ai/claude-code=       | Minified; uses [[https://github.com/shcv/astdiff][astdiff]] for diffs         |
| =claude-agent-sdk= | npm =@anthropic-ai/claude-agent-sdk= | Extracts both =cli.js= and =sdk.mjs=      |
| =codex=          | GitHub =openai/codex= (=rust-v*= tags)| Rust source under =codex-rs/=, plain diff |

Project definitions live in the =PROJECTS= dict near the top of =tools/sync.py=.

** One-Time Setup

1. Python deps:
   #+begin_src bash
   pip install requests packaging python-dotenv claude-agent-sdk
   #+end_src

2. Node tools (for npm-based projects):
   #+begin_src bash
   npm install -g prettier
   #+end_src

3. Optional but recommended for Claude Code / Agent SDK diffs:
   #+begin_src bash
   # https://github.com/shcv/astdiff — semantic JS-aware diff
   cargo install --git https://github.com/shcv/astdiff
   #+end_src

4. Authenticate the harness used for changelog generation. =sync.py= calls
   =claude-agent-sdk='s =query()= against an isolated config dir at
   =~/.claude/projects/harness-investigations/= so its sessions don't pollute
   =--resume= and ccusage groups them under that name. =.credentials.json=
   and =settings.json= are symlinked in from =~/.claude= on first run.

5. Discord posting (optional). Copy the example config and edit:
   #+begin_src bash
   cp tools/config.py.example tools/config.py
   #+end_src
   Webhook URLs go in =config.py= (gitignored). Each webhook subscribes to one
   or more channels (=claude-code=, =codex=, ...).

** Running

=sync.py= takes =--project= plus phase flags. The phases run in order;
=--all= turns on every phase end-to-end.

#+begin_src bash
# Just download new releases for the default project (claude-code)
python tools/sync.py

# Full pipeline for one project
python tools/sync.py --project codex --all

# Latest release only
python tools/sync.py --project claude-code --all --latest

# Backfill from a specific version
python tools/sync.py --project claude-code --changelog --since v2.0.0

# Newest first (instead of chronological)
python tools/sync.py --project codex --all --new-first

# Redo a single version's changelog (creates iteration -2, -3, ...)
python tools/sync.py --project claude-code --redo 2.0.37 --changelog

# Discord post only (no regeneration)
python tools/sync.py --project codex --post
#+end_src

** Phase Flags

| Flag           | What it does                                                              |
|----------------+---------------------------------------------------------------------------|
| (none)         | Download missing originals into =archive/<proj>/original/= or =source/=   |
| =--prettier=   | Run prettier over downloaded files (npm projects only)                    |
| =--diff=       | Generate diffs between consecutive versions                               |
| =--changes=    | Filter diff noise (version bumps, build paths, reformatting)              |
| =--changelog=  | Ask the LLM for a changelog; implies =--changes=                          |
| =--cleanup=    | Strip duplicate headers / meta-commentary from generated changelogs       |
| =--post=       | Post new changelogs to Discord via webhook                                |
| =--all=        | Enable all of the above                                                   |

Useful modifiers: =--latest=, =--since VERSION=, =--new-first=, =--redo SPEC=,
=--annotate= (Haiku pre-pass; expensive), =--astdiff-threads N=, =--dry-run=.

** What Lands in Git

Only =archive/<project>/changelog/= (which contains the system prompt and the
generated changelogs). Everything else — originals, pretty files, diffs,
changes, source clones, =.sync-temp-*=, =.env=, =tools/config.py=,
=.last_posted_version_*= — is gitignored.

** Customizing the System Prompt

The first =--changelog= run for a project writes a default system prompt to
=archive/<project>/changelog/system-prompt.md=. Edit it to tune tone, depth, or
focus; subsequent runs read the file you committed.

4. Open a PR.
